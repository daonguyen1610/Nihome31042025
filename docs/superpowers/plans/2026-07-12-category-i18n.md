# Category i18n Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Localize activity/news/project category names on the public site, add Category to the admin "Content Translation" tab, and fix every category write path that was silently skipping `NameVi`/the category FK — without touching the content APIs' `Category` field (which the admin edit forms depend on for their string-match round-trip).

**Architecture:** Backend keeps categories on their existing fixed-column pattern (`NameVi/NameEn/NameZh/NameJa` on `ActivityCategory`/`NewsCategory`/`ProjectCategory`); no new tables, no response-shape changes to `Activity`/`News`/`Project` APIs. The Content Translation tab gets a special-cased branch in `TranslationsController` that reads/writes those fixed columns directly while presenting Category as a single `Name` field per language, matching the shape the already-generic frontend expects. Public-site localization is a pure frontend change: join each item's `categoryId` against the existing (already lang-agnostic) category-list endpoints and reuse the existing `localizedName()` helper.

**Tech Stack:** ASP.NET Core 8 / EF Core (SQL Server) backend, xUnit + Moq + EF InMemory for backend tests, React + TypeScript + TanStack-Query-less custom hooks frontend.

```csharp
    public async Task<(int? Id, string Name)> ResolveAsync(int? categoryId, string? categoryName)
    {
        if (categoryId.HasValue)
        {
            var byId = await db.ActivityCategories.FindAsync(categoryId.Value);
            if (byId == null)
            {
                throw new InvalidOperationException("Danh mục bài đăng không tồn tại.");
            }
            return (byId.Id, byId.Name);
        }

        var trimmed = (categoryName ?? string.Empty).Trim();
        if (string.IsNullOrWhiteSpace(trimmed))
        {
            return (null, string.Empty);
        }

        var existing = await db.ActivityCategories
            .FirstOrDefaultAsync(c => c.Name.ToLower() == trimmed.ToLower());

        if (existing != null)
        {
            return (existing.Id, existing.Name);
        }

        var maxSortOrder = await db.ActivityCategories
            .AsNoTracking()
            .Select(c => (int?)c.SortOrder)
            .MaxAsync() ?? 0;

        var created = new ActivityCategory
        {
            Name = trimmed,
            NameVi = trimmed,
            IsActive = true,
            SortOrder = maxSortOrder + 1,
        };
        db.ActivityCategories.Add(created);
        await db.SaveChangesAsync();
        logger.LogInformation("Auto-created activity category {CategoryName} from activity payload", trimmed);
        return (created.Id, created.Name);
    }
```
